# Hexagonal Architectural Decisions (ADR)

Status: Not implemented yet.

## Objective

Capture architecture decisions that preserve core-domain isolation and adapter replaceability.

## Decision Topics

1. Domain core boundaries and use-case orchestration
2. Inbound/outbound port definitions
3. Adapter implementation patterns
4. Dependency inversion and module rules
5. Testing strategy by layer (core, port, adapter)

## ADR Requirements

Each ADR must include context, decision, alternatives, trade-offs, consequences, and linked requirement IDs.

## Output Artifacts

- Hexagonal ADR set
- Dependency rule specification
- Requirement-to-ADR mapping
